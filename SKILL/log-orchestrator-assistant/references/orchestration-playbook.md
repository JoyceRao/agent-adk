# Orchestration Playbook

## 标准流程

1. 收集输入参数：日志路径、时间窗、类型、关键词、源码目录。
2. 先筛选再分析：避免全量日志直接深度解析。
3. 根据分析结果输出 CRISP-L 报告并落盘。

## 快捷分流

- 仅筛选：返回筛选统计与样本预览。
- 仅分析：返回结构化分析与源码关联证据。
- 仅报告：优先走一键生成工具。

## 失败回退

- `matched_entries = 0`：提示扩窗或放宽关键词。
- `source_correlations = 0`：提示检查 `source_root` 或日志定位格式。
- 报告写入失败：先返回报告正文，再提示文件系统问题。
