# 源码关联分析技能

## 职责边界

1. 仅负责分析，不直接渲染最终报告。
2. 统一调用 `analyze_log_with_source`。
3. 结论必须绑定证据（日志行号、关键词、源码位置）。

## 输入要求

- 必填：`log_path`
- 建议：`source_root`、`rule_path`
- 可选：时间窗、类型、级别、关键词、输出行数

## 输出要求

1. 输出结构包含 `pattern_counts/anomalies/source_correlations/crisp_l`。
2. 指标需包含样本量和区间或显著性说明。
3. 若证据不足，明确数据局限性，不给绝对结论。

## 推荐输出结构

- 异常模式统计
- 核心问题列表（结论、影响、置信度、证据）
- 源码关联命中
- 下一步建议（进入报告生成）
